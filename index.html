Spring / JPA / Hibernate¶

Cours de 3ième année EPSI Bordeaux : nous allons voir comment développer des applications Web qui interagissent avec des bases de données.

Dans un premier temps nous verrons comment interagir avec des bases de données relationnelles en utilisant le standard JPA (Java Persistence API) et Hibernate. Avec ces technologies, nous aurons l’occasion de découvrir ou de mieux comprendre ce qu’est un ORM (Object Relationnal Database).

Ensuite nous utiliserons le Spring Framework et Spring Boot pour concevoir des applications Web. Cette partie sera l’occasion d’aborder plusieurs points fondamentaux dans d’architecture logicielle :

  • les notions d’inversion de contrôle et d’inversion de dépendance,

  • la mise en pratique du modèle MVC pour le développement d’applications Web,

  • la conception d’applications multi-couches (N-Tiers) avec les notions de contrôleur, de service et de repository.

Exercice¶

TP à rendre - Les animaux

Projet initial

Vous devez télécharger le projet initial animaux-orm.zip.

Ce projet contient la classe com.animoz.Application fournissant un exemple d’initialisation de JPA. Vous trouverez le fichier src/main/sql/database.sql qui vous permettra de créer le schéma de base de données dans MySQL ou MariaDB. Vous trouverez également le fichier src/main/resources/META-INF/persistence.xml vous permettant de configurer l’accès à la base de données (pilote JDBC, URL d’accès à la base de données, nom d’utilisateur et mot de passe).

Programmes à implémenter

Pour chaque fonctionnalité ci-dessous, créez une classe Java sur le modèle de la classe com.animoz.Application. Vous devrez également créer les classes pour représenter les entités.

  1. Demander à l’utilisateur de saisir le nom d’une espèce et créer cette espèce en base de données :

    Scanner in = new Scanner(System.in);
System.out.print("Nom de l'espèce : ");
String nom = in.nextLine();

  2. Demander à l’utilisateur de saisir l’identifiant d’un animal et afficher les informations de l’animal :

    Scanner in = new Scanner(System.in);
System.out.print("Id animal : ");
long id = in.nextLong();

  3. Afficher sur la console la liste des espèces.

  4. Afficher sur la console la liste des espèces triée par nom.

  5. Afficher sur la console la liste des animaux carnivores

  6. Demander à l’utilisateur de saisir le nom d’un animal et afficher l’animal (s’il existe). Essayer de faire une recherche indépendante de la casse du nom.

    Scanner in = new Scanner(System.in);
System.out.print("Nom de animal recherché : ");
String nom = in.nextLine();

  7. Afficher sur la console le nombre total de soigneurs.

  8. Afficher sur la console les informations sur le dernier soigneur recruté.

  9. Afficher sur la console la liste des animaux de l’espèce Artiodactyla.

  10. Demander à l’utilisateur de saisir le nom d’un animal puis demander à l’utilisateur de saisir un nombre d’individus pour une population. Créer une nouvelle population pour cet animal avec le nombre d’individus saisi.

  11. Demander à l’utilisateur de saisir le nom d’un animal. Afficher le nombre total d’individus de cet animal. Utiliser une requête JPQL pour calculer directement ce nombre.

  12. Demander à l’utilisateur de saisir le nom d’une espèce. Afficher le nombre total d’individus de cet espèce. Utiliser une requête JPQL pour calculer directement ce nombre.

Attention

Lorsque vous devez effectuer des requêtes, ne faites pas de requêtes SQL, utilisez JPQL.

Pour le language JPQL, reportez-vous au Wikibook.

Exercice¶

TP à rendre - Les animaux

Projet initial

Vous devez télécharger le projet suivant animaux-spring.zip et réaliser les développements à partir de ce projet.

Ce projet contient la classe com.animoz.AnimozApplication pour lancer l’application Spring Boot. Pensez à modifier le fichier de configuration src/main/resources/application.properties pour mettre à jour les paramètres de connexion à la base de données.

Cette application utilisera la même base de données que celle mise en place pour les exercices avec JPA. Vous pouvez ajouter les classes d’entités dans ce projet car elles vont nous être utiles.

Fonctionnalités à implémenter

Prévoyez un menu HTML dans l’application pour accéder à la section soigneurs et à la section animaux.

  1. Afficher la liste des soigneurs (par ordre alphabétique). On affiche dans la liste le nom, le numéro et la date de recrutement.

  2. Afficher la liste des animaux (par ordre alphabétique). On affiche dans la liste, le nom, le nom de l’espèce et le régime alimentaire.

  3. Créer une formulaire de recherche pour rechercher un animal par son nom (prévoir une recherche de type like). Le formulaire de recherche sera visible dans la page affichant la liste des animaux.

  4. Ajouter un soigneur. On pourra ajouter un soigneur à partir d’une page spécifique. Pour ajouter un soigneur, on saisit son nom, son numéro et sa date de recrutement. Tous les champs sont obligatoires. La date de recrutement ne peut pas être dans le futur.

  5. Ajouter un animal. On pourra ajouter un animal à partir d’une page spécifique. Pour ajouter un animal, il faut saisir son nom, sa description, son régime alimentaire, son espèce. Seul le champ description est optionnel. Si un animal avec le même nom existe déjà dans la base de données, la création ne doit pas aboutir.

  6. Afficher le détail d’un animal. Le détail d’un animal est visible en cliquant sur la ligne de l’animal dans la liste. On affiche le nom, le nom de l’espèce, le régime alimentaire, la description et la liste des soigneurs.

  7. Depuis l’écran de détail d’un animal, on doit pouvoir modifier la liste des soigneurs. Dès qu’un animal a au moins un soigneur, il n’est pas possible d’enlever tous les soigneurs de la liste : un animal doit avoir au moins un soigneur.

Pour réaliser ces fonctionnalités, vous respecterez le modèle MVC ainsi qu’un modèle de développement en couche.

Il n’est pas demandé de réaliser des traitements sur le poste client en JavaScript, mais toutes les validations doivent être réalisées sur le serveur.
